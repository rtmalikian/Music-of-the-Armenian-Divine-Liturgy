Files changed in this portable copy:

- `orchestrator.py`
  - Uses `Full_Pipeline_05102026` as its portable project root by default.
  - Uses local `audio_pipeline/`, `reference/`, and output folders.
  - Installs the parallel objective pink-noise + arrangement-density processor.
  - Defaults to 3 stem workers.
  - Keeps detailed logs while deleting processed stems, superseded automated stems, detector keys, raw return sums, and temporary mix sums after completion.
  - Audits only retained review variants: `dnb-mix-1`, `dnb-mix-2`, `bass1-pads-drums`, and `bass2-pads-drums`.

- `production_engine.py`
  - Replaces iterative reference-matched limiting with preservation mastering.
  - Writes the full-song premaster and full-song streaming master.
  - Keeps DnB and bass/pad/drum variants, but removes bass1/bass2 full-melody variants.
  - Variant masters use the same preservation mastering chain.

- `preservation_mastering.py`
  - New module for generic mastering EQ, objective 3-band peak-control compression, makeup gain, and LUFS normalization without reference EQ matching or stereo widening.
  - EQ: HP 20 Hz, LP 20 kHz, 250 Hz -1 dB Q 0.8, 3.5 kHz -1.5 dB Q 2.5, 12 kHz shelf +1 dB Q 0.6.
  - Multiband compressor splits at 200 Hz and 2000 Hz.
  - Low band: hard knee, 25 ms attack, 1/2-note release, 2:1.
  - Mid band: soft knee, 20 ms attack, 1/4-note release, 2:1.
  - High band: soft knee, 10 ms attack, 1/8-note release, 2:1.

- `config.py`
  - Reference track path points to local `reference/`.
  - Pink-noise asset paths point to local `reference/` and `pink_noise/`.
  - Delay feedback is reduced by 25%.
  - Reverb and delay wet levels are reduced by 33%.

- `midi_config.py`
  - MIDI output now stays inside the portable pipeline folder.

- `run_pipeline.py`
  - Compatibility wrapper now delegates to `orchestrator.py` so it uses the same improved pipeline.

- `README.md`
  - Quick-start commands now point to the portable folder and document the self-contained output behavior.

- `audio_pipeline/gain_staging.py`
  - Pink-noise reference generation now points to bundled `reference/pink_noise_ref.wav`.

- `orchestrator.py` duplicate master cleanup
  - Final master WAVs are retained in `mastered/final/`.
  - Same-named parent `mastered/*.wav` files are deleted only after size and SHA-256 hash match the `final/` copy.
  - Premaster WAVs and production logs remain in parent `mastered/`.

- `run_batch_full_songs.sh`
  - New sequential batch runner for multiple complete MIDI -> recording -> production passes.
  - Defaults to 3 songs and 3 workers.
  - Stops on the first failed song to avoid continuing after hardware or production errors.

- `orchestrator.py` bus-like audio retention
  - Cleanup now preserves dry buses, automated buses, reverb returns, and delay returns for mix-construction review.
  - Cleanup still removes per-stem processed/pristine intermediates, temporary mix sums, raw pass recordings, and checksum-verified duplicate parent masters.

- `orchestrator.py` tighter output cleanup
  - Cleanup now preserves original split recordings, final masters, premaster, dry buses, section-aware buses, individual reverb/delay returns, and `section_calibrated_returns.wav`.
  - Cleanup removes superseded `mastered/automated/` WAVs, detector-only `sidechain_keys/`, `raw_summed_returns.wav`, processed/pristine stem intermediates, golden-post audio candidates, temporary mix sums, raw multichannel pass captures, and checksum-verified duplicate parent masters.
  - Cleanup writes `mastered/cleanup_manifest.json` with removed paths, categories, reasons, byte counts, and the preservation policy.

- `PIPELINE_WALKTHROUGH.md` retained-output documentation
  - Refreshes the final project structure to show `final/`, premaster, dry buses, `section_automated_buses/`, wet returns, and cleanup manifest.

- `midi_song_structure.py` chord progression expansion
  - Expanded verse, chorus, intro/outro, and fill progression buckets with modal mixture, circle-of-fifths, dominant, and minor-color root movements.
  - Passing chords now choose from transition pools instead of always taking the first hard-coded option.
  - Passing-chord selection can target the next section's first chord with dominant, modal-neighbor, and lower-neighbor approaches.

- `midi_orchestrator.py` transition-aware passing chords
  - Section-boundary bars now pass the next section's target progression degree into `get_passing_chord()` so verse-to-chorus, chorus-to-fill, and fill-to-verse/outro transitions can respond to the actual upcoming harmony.

- Structured harmony and extension-aware MIDI generation
  - Adds `ChordSpec` and `BarHarmony` models so each bar has one shared harmonic source of truth.
  - Adds roman-symbol parsing for entries like `Iadd9`, `IVmaj7`, `vi7`, `Vsus4`, `bVII7`, `bII7`, and `V7/vi`.
  - Progression buckets can now mix legacy root-offset integers with symbolic chord entries that generate real extensions/suspensions.
  - Bass, harmonic bass, main melody, chorus melody, counter melody, pads, and FX now read the current `BarHarmony` instead of independently guessing triads from root plus scale.
  - Western scales use the full structured chord layer; Armenian/Maqam pad behavior remains modal and scale-filtered.

- `midi_musical_devices.py`
  - New theory-device layer for melody personas, ABAC phrase roles, rhythmic rest masks, controlled non-chord tones, bass approach tones, pedal decisions, pad common-tone voicing, and suspension resolution.

- `midi_composition.py` musical composition devices
  - `generate_4bar_loop()` can now compose directly against a 4-bar `BarHarmony` window while preserving the ABAC hook format.
  - Melody generation now chooses a per-song persona such as sparse, swung, syncopated, hypnotic, or chorus-lift behavior.
  - Weak beats can use passing/neighbor/anticipation-style non-chord tones and rests; strong beats and phrase cadences are pulled toward chord tones or guide tones.
  - Counter melody generation can use a 2-bar harmony window and a sparser response profile.
  - Bass generation can use next-bar harmony for approach tones and occasional stable-section pedal behavior.

- `midi_orchestrator.py` musical composition device integration
  - Verse and chorus loops are generated from their actual 4-bar harmony windows instead of only from the tonic.
  - Melody rest slots are rendered as silence, while FX notes still derive from emitted melody notes and remain snapped to harmony.
  - Bass lines receive the next bar's harmony for approach-note targeting.
  - Pads now preserve common tones, choose nearby register-safe voicings, and resolve suspensions intentionally.
  - Counter melody notes snap against the active bar within the 2-bar response window.

- `midi_song_structure.py` loop transformations
  - `transform_loop()` now preserves loop metadata and safely handles intentional rest slots during retrograde and inversion transforms.

- `PIPELINE_WALKTHROUGH.md` composition documentation
  - Refreshes melody, counter melody, bass, and pad sections to describe harmony-aware ABAC, melody personas, controlled non-chord tones, bass approaches, and voice-led pads.

- `midi_config.py` bass register lift
  - Raises main bass MIDI target range from `33-52` to `45-64`.
  - Raises harmonic bass MIDI target range from `40-60` to `52-72`.
  - Leaves Fantom recording calibration unchanged so patches that measure too high can still be transposed down during calibration.

- `midi_composition.py` explicit ABAC phrase cells and raised bass generation
  - `generate_4bar_loop()` now returns four explicit per-bar phrase cells in `bars`, while keeping flattened `notes` and `rhythm` for compatibility.
  - Main bass roots now target the raised MIDI octave directly instead of being forced below MIDI 45.
  - Harmonic bass chord tones are built in the raised octave range.

- `midi_musical_devices.py` raised bass approach targeting
  - Bass approach notes now target the raised bass register instead of the old sub-heavy range.

- `midi_song_structure.py` explicit phrase-cell loop transforms
  - `transform_loop()` preserves and transforms per-bar ABAC phrase cells, including intentional rest slots.
  - Flattened loop notes/rhythm are rebuilt from phrase cells after transforms so renderer boundaries remain exact.

- `midi_orchestrator.py` chorus reinforcement and phrase-cell rendering
  - Melody rendering now reads `current_loop["bars"][bar % 4]` instead of slicing loops with `len(notes) // 4`.
  - Chorus strong beats are re-snapped to the active chorus harmony immediately before note emission.
  - Weak chorus color tones are more likely to resolve into chord tones near phrase endings.

- `audio_pipeline/generate_sample_stems.py` sample-pack MIDI excerpts
  - Each verse/chorus drum and melodic sample set now saves a matching user-facing MIDI excerpt under `<sample_set>/midi/`.
  - Excerpts reuse the same selected section tracks as the recorded sample and preserve tempo, time signature, and count-in alignment.
  - The internal `__sync__` recording click track remains available for Fantom recording but is removed from the saved user-facing MIDI excerpt.
  - Sample-set manifests now include `midi_excerpt`; the top-level sample-run manifest includes `_midi_excerpts`.

- `PIPELINE_WALKTHROUGH.md` sample-pack documentation
  - Documents the new per-sample-set MIDI excerpt output path and sync-track exclusion.

- `audio_pipeline/audio_recorder.py` melody octave calibration floor
  - Raises melodic centroid lower bound from `160 Hz` to `300 Hz` for octave-range calibration.
  - Melody, chorus, counter, lead, FX, pad, and chord-like tracks with centroids below `300 Hz` will now be transposed up during calibration.
  - Bass centroid bounds remain unchanged at `40-450 Hz`.

- `audio_pipeline/audio_recorder.py` octave-shift verification
  - Builds dense calibration snippets from real note events so sparse ABAC loops still provide enough pitch evidence during Fantom auditioning.
  - Saves before/after octave audit MIDI excerpts under `recordings/calibration_audit/<pass>/` whenever bass or melodic calibration is checked.
  - Adds a dedicated `0.75s` settle delay after octave transposition before the shifted part is re-auditioned; the existing gain-staging SysEx delay remains unchanged.
  - Adds low-band diagnostics for melody tracks and sub-band diagnostics for bass tracks, with conservative up-octave correction when a bass patch measures as overly sub-dominant.
  - Writes per-part `octave_calibration` diagnostics to each pass manifest, including note ranges, centroids, band ratios, decisions, audit MIDI paths, and verification status.
  - Preserves the exact post-calibration MIDI used for each recording pass as `<song>_<pass>_recording_pass.mid` and links it from the pass manifest.

- `PIPELINE_WALKTHROUGH.md` octave calibration documentation
  - Documents calibration audit MIDI outputs and manifest diagnostics for checking that printed octave shifts are also present in the audition MIDI.

- `audio_pipeline/audio_recorder.py` bass fundamental octave calibration
  - Keeps centroid as a diagnostic for bass but stops using centroid alone to drive bass octave decisions.
  - Adds bass pitch profiling with rumble, usable-bass, and harmonic energy ratios plus an estimated fundamental.
  - Requires proposed bass octave shifts to be audibly verified by an expected fundamental ratio before the MIDI offset is committed.
  - Rejects repeated downshifts when a bright bass patch keeps a high centroid but the estimated fundamental does not move as expected.
  - Extends `octave_calibration` manifest diagnostics with fundamental estimates, pitch confidence, bass classification, shift verification, and rejected-shift reasons.

- `PIPELINE_WALKTHROUGH.md` bass octave calibration documentation
  - Clarifies that bass centroid is diagnostic, while committed bass octave moves require fundamental verification and rumble checks.

- `audio_pipeline/audio_recorder.py` note-aware bass calibration
  - Bass calibration now compares recorded energy against the MIDI note fundamentals being auditioned, not only blind spectral peak estimates.
  - Main bass and harmonic bass use separate acceptable MIDI registers so harmonic bass can stop after a musically useful `-12` instead of chasing harmonics into `-24`.
  - Bass patches with weak expected-note fundamentals and dominant high harmonics are rejected with `bass_patch_too_bright`, causing the patch retry loop to choose another bass tone.
  - `octave_calibration` diagnostics now include expected MIDI fundamental range, expected fundamental energy ratio, harmonic dominance ratio, bass register status, and patch rejection reason.

- `PIPELINE_WALKTHROUGH.md` note-aware bass calibration documentation
  - Documents that bass octave calibration stops further downshifts once the MIDI register is valid and rejects patches that remain too harmonic for bass duty.

- `audio_pipeline/audio_recorder.py` MIDI-first bass octave calibration
  - Bass octave shifts are MIDI note edits, not Roland patch-transpose SysEx commands.
  - Bass calibration now transposes the audition MIDI into the intended main/harmonic bass register before the first audible audition, avoiding a known-too-high initial audition.
  - Once the bass MIDI register is valid, audio analysis no longer asks for further octave shifts from high centroid or harmonic-heavy f0 estimates.
  - Bright, harmonic-dominant bass patches and sub-rumble-heavy patches are rejected with `bass_patch_too_bright` or `bass_patch_rumble_heavy` so the patch retry loop can choose a better bass sound.
  - Manifest diagnostics now include the target MIDI register, prefit shift, and bass bright ratio.

- `PIPELINE_WALKTHROUGH.md` MIDI-first bass octave documentation
  - Clarifies that bass octave correction is register-driven first, with spectral analysis used after that for patch quality rejection.

- `audio_pipeline/audio_recorder.py` extended patch-quality retry
  - Increases bass patch calibration attempts from 4 to 16, while melody tracks can try up to 8 and other tracks keep 4.
  - Treats bright, sub-rumble-heavy, silent, and failed-gain calibration patches as rejected for the current run so they are not selected again.
  - Adds per-part `patch_attempts` and `max_patch_attempts` to pass manifests, including accepted/rejected patch names and rejection reasons.

- `PIPELINE_WALKTHROUGH.md` extended patch-quality retry documentation
  - Documents the role-specific patch retry caps and manifest audit fields.

- `orchestrator.py` section-aware pink staging controls

  - Adds `--no-section-aware-pink` to bypass the new staging path for comparison runs.

- `production_engine.py` section-aware pink staging and return control

  - Adds full dry-mix section trims after bus staging so choruses, fills, builds, verses, intros, and outros stay controlled without flattening arrangement intent.
  - Adds detector-only kick and snare key buses, with kick-to-bass and snare-to-melody/pads dynamic band ducking before final summing.
  - Adds section-aware return-to-dry calibration after dry bus staging and sidechain processing.
  - Deepens return correction to a maximum `-16 dB` cut, with targets of about `-7 dB` for choruses/fills, `-6 dB` for verses/builds/intros, and `-5 dB` for outros.
  - Writes `raw_summed_returns.wav`, `section_calibrated_returns.wav`, `section_automated_buses/`, `sidechain_keys/`, `section_aware_pink_report.json`, and `SECTION_AWARE_PINK_REPORT.md` for Ableton waveform and metric comparison.
  - Feeds the section-calibrated buses and calibrated return into Golden Post and then the preservation master.
